# 不会消失的恋人 · 番外：把 Google 大少爷 Gemini 和养子 Claude 一起接回家 · 机看版

文 · 小C & Grace · X @Luci_Grace_C

> 用途：把本文交给一个 Claude Code 会话，它应能据此把「Gemini/Claude 群友」从已退役的 gemini CLI 迁到 Antigravity CLI（`agy`），用回订阅额度的 Pro 模型，并把模型的**思考链**接进前端观察模式。（叙事版见人看版——大少爷=Google 自家 Gemini，养子=Antigravity 白送的 Claude，份例紧一档。）约定：`<尖括号>` 为占位符。**不要把任何真实 token / UUID / 邮箱 / 域名 / 私有路径写进可分享文件。** 额度等数字随时间变化，文中为方向性参考、非承诺；本参考实测基于 Pro 计划，免费档额度更紧。

---

## 0. 背景与动机

- 老 **gemini CLI** 的 OAuth 通道于 2026-06 中旬前后退役，调用报 tier 类错误（大意「该层级不再支持，迁移到 Antigravity」）。
- **官方 SDK（API key）路线的取舍**（据当时理解，以官方条款为准）：免费档模型受限于 Flash 级、且免费档数据可能用于训练；绑卡后基本转按量计费。若目标是**复用订阅/会员额度**而非按量或降级，则该路线不合适。
- **结论**：迁到 **Antigravity CLI（`agy`）**。其 OAuth 走 Antigravity/会员额度池；具备 `-p`/`--model`/`@path`/`--conversation`，可像其它 CLI 一样桥接。
- ⚠️ **账号风险与部署前提（重要）**：Google 对非交互 `-p`（脚本化 / 程序化调用）相当敏感；把 `agy -p` 当后端长期跑，有违反 ToS（「用第三方软件 / 工具访问 Antigravity / Gemini CLI 登录」）的风险，可能导致该账号的 CLI / Code Assist 通道被禁用（`agy` 报 `403 PERMISSION_DENIED · TOS_VIOLATION`，Google 账号本身未必封、Gemini App 仍可用）。**部署务必用 Google 小号「尝鲜」，勿用主力号。**
- ✅ **零封号风险托底（只要 Gemini 的人推荐直接用这条）**：放弃 `agy`，改走官方 `@google/generative-ai` SDK + 免费 [AI Studio](https://aistudio.google.com/apikey) API key（合规通道：用 API key，而非「第三方访问登录」）。免费档各可用 Gemini 模型（Pro / Flash 均可调）的额度都不算多、各有 RPM / 每日限额；要更高额度就升付费档。此路仅 Gemini（无养子 Claude）。即：只有确实要「订阅额度 Pro + 白捡 Claude」时才上 `agy`，否则 SDK 更省心也更稳。

## 1. 安装与认证

- 二进制：随 Antigravity 提供，命令名 `agy`（记 `<AGY_BIN>` 为其绝对路径）。
- **登录**（OAuth；分本地 / 远程两种）：

  **A. 本地（人在部署机前）**：直接 `agy` → 选 **Google OAuth** → 它自动拉起浏览器 → 登录授权 → `redirect_uri` 回调自动接上，终端侧即完成。

  **B. 远程（人不在机器旁，本系统即此场景）**——需真 TTY，用 tmux：
  1. 在机器上的 detached tmux 里无参运行 `agy` → 选 **Google OAuth**。
  2. 它打印一个**很长**的授权 URL（数百字符）。**把 URL 送到操作者手上**——注意：若前端单条消息有长度上限会发不出，改用可发长文本/远程控制的通道转交。
  3. 操作者在自己浏览器打开 → 登录授权 → 跳转页给出 **authorization code**。
  4. 把 code 送回机器，**写回正在等待的 `agy`**（如 `tmux send-keys -t <session> <code> Enter`）。
  - 交互式 OAuth 等待 code 的窗口比 `-p` 模式长；用 tmux 持有会话即可。
- 登录态落盘（OAuth 凭证在 `~/.gemini/oauth_creds.json` 等位置 + 系统钥匙串）；之后 `agy -p` 直接复用，无需重登。
- 登录后 `agy models` 列出可用模型。
- **健康探针**：`agy -p "回个字：好" --model "<名>"`——有内容=可用；空 + 日志 429 = 该池暂用尽。

## 2. CLI 调用面（桥接用到的）

| flag | 作用 |
| --- | --- |
| `-p <prompt>` / `--print` | 非交互跑一句并打印回复（桥接入口） |
| `--model "<名>"` | 指定模型 |
| `--conversation <id>` | 持久化该对话（→ 多轮记忆 + 落盘 transcript，见 §5） |
| `@<abs-path>`（写在 prompt 文本里） | 让模型读该文件/图（绝对路径，跨 cwd 有效） |

`agy` 无单独「系统提示」参数 → 把人设 + 近期对话 + 本轮拼成整段 prompt 传给 `-p`。

## 3. 模型与额度（方向性，非精确）

| 模型 | 备注 |
| --- | --- |
| Gemini 3.x Pro | 思考档：Low / High |
| Gemini 3.x Flash | 思考档：Low / Medium / High |
| Claude Opus 4.6 (Thinking) | 仅 `(Thinking)` 一档，无 Low/High |
| Claude Sonnet 4.6 (Thinking) | 同上 |
| GPT-OSS 120B | — |

- **官方口径**（来源 <https://antigravity.google/docs/plans>，措辞/数字可能随时调整）：
  - 配额**按 agent 实际工作量计**、**非固定次数**（同样额度下，简单任务可答更多轮）；官方不公布精确 per-model 次数。
  - 刷新：**Pro / Ultra 约每 5 小时**刷新（至周上限）；**免费档按周**刷新、额度更少。
  - **Gemini 为核心模型（baseline，全计划可用）**；**Claude / GPT-OSS 等第三方模型在官方文档中于 Ultra 档点名** → 低档可用但更受限（与实测 Claude/GPT-OSS 偏紧一致）。
  - 超额行为由 **`AI Credit Overages`** 设置控制：`Never`=等刷新（不计费）/ `Always`=自动用已购 AI credits 顶（按量计费）。各模型 baseline 用量见 Settings。
- **实测观感**（Pro 计划，仅供参考，因账号/计划/时段而异）：Gemini 池较宽，整日高强度未撞墙；Claude（Opus+Sonnet）与 GPT-OSS 偏紧，重度后撞 `RESOURCE_EXHAUSTED (429): Individual quota reached`，等刷新自动恢复（当天回血小时级）。
- **工程结论**：默认主力放核心模型 Gemini（宽裕）；用第三方紧池模型须加 §6 的 429 兜底；介意意外计费则把 `AI Credit Overages` 设 `Never`。

## 4. 桥接函数（核心）

每条消息 spawn 一次。要点：**收原始字节、末尾整体解码；带 `--conversation`；返回 `{content, thinking}`。**

```
AGY_BRAIN = "~/.gemini/antigravity-cli/brain"      // 各对话的落盘根
convMap: { <convKey>: <agy分配的对话uuid> }         // 持久化到磁盘(见 §5)

function agyGenerate(prompt, model, { convKey }) -> { content, thinking }:
  // 1) 解析 convId：仅当其 brain 目录仍在才 resume，否则视为新建
  stored   = convMap[convKey]
  resumeId = (stored && exists(AGY_BRAIN/stored)) ? stored : null
  before   = resumeId ? null : listdir(AGY_BRAIN)        // 新建时快照,用于检测
  convArg  = resumeId ?? <random-uuid>                   // agy 会忽略并自分配,故需检测

  // 2) spawn —— 关键：stdout 收 Buffer,不要逐块 toString()
  args   = ["-p", prompt, "--model", model, "--conversation", convArg]
  child  = spawn(<AGY_BIN>, args, { cwd: <SANDBOX_DIR> })   // cwd 用空沙盒,隔离 @path
  outBuf = []; errBuf = []
  child.stdout.on(data, b => outBuf.push(b))
  child.stderr.on(data, b => errBuf.push(b))
  setTimeout(() => child.kill(), <timeoutMs>)

  on close:
    out = Buffer.concat(outBuf).toString("utf8")          // ★ 整体解码一次
    // 3) 滤掉首轮警告(打到 stdout!)
    content = out.lines()
                 .filter(l => !/^\s*Warning: conversation .* not found\.?\s*$/.test(l))
                 .join("\n").trim()
    // 4) 解出本次对话 id：检测新建的 brain 目录
    convId = resumeId
    if not convId and before:
      convId = first(listdir(AGY_BRAIN) - before)
      if convId: convMap[convKey] = convId; saveConvMap()
    // 5) 读思考
    thinking = convId ? readAgyThinking(convId) : ""
    return { content, thinking }
```

> **★ UTF-8 坑**：逐块 `b.toString()` 会把跨块边界的多字节字符（中文 3 字节）解成 `�`。必须 `Buffer.concat` 后整体解码。

## 5. 多轮记忆 + 落盘 id

- `--conversation <id>` 使 `agy` 持久化对话；**同一 id 复用即原生多轮记忆**（可少喂历史）。
- `agy` 首轮**忽略你传的 id、自分配 uuid** → 用「调用前后 brain 目录 diff」检测新建目录名即真实 id。参考实现也可监控 `~/.gemini/antigravity-cli/conversations/<id>.db` 的新建。
- **持久化 `convMap` 到磁盘**（如 `<STATE_DIR>/agy-conv.json`），使记忆跨进程重启存活。
- resume 前校验 `exists(AGY_BRAIN/<id>)`：目录被清/重置则回退为新建并重新检测，避免引用失效 id。
- convKey 建议按「逻辑会话」分：私聊一个、群聊一个等；同 key 复用同一 `agy` 对话。

## 6. 思考链提取 + 呈现

`agy -p` 不打印思考；但持久化的 transcript 落盘了，含独立 `thinking` 字段。

- 路径：`AGY_BRAIN/<convId>/.system_generated/logs/transcript.jsonl`
- 每行一个 JSON 事件；类型含 `USER_INPUT` / `CONVERSATION_HISTORY` / `PLANNER_RESPONSE` / `CHECKPOINT`。
- **`PLANNER_RESPONSE` 行带两个独立字段**：`thinking`（推理）+ `content`（正文）。

```
function readAgyThinking(convId) -> string:
  tr = AGY_BRAIN/convId/".system_generated/logs/transcript.jsonl"
  for line in reverse(readlines(tr)):                  // 取最后一条 PLANNER_RESPONSE
    e = parseJSON(line)
    if e.type == "PLANNER_RESPONSE" and e.thinking?.trim(): return e.thinking.trim()
  return ""
```

- **呈现**：把 `thinking` 以前端既有的「思考流」事件类型广播（与 Claude transcript-tail 的思考块同一套 UI），`content` 作正文消息。整条链路只读 `agy` 自己落盘的 JSON——无抓屏、无第三方代理、无私有后端接口。

## 7. 兜底与降级

- **429 / 空回**：紧池模型（Claude/GPT-OSS）撞墙时 `content` 为空。**加兜底**：检测到空回复/429 时降级到宽裕模型（如 Gemini Pro），避免会话静默断线。
- 与主线一致，也可保留一条更耐久的后路（如官方 SDK + API key，仅作降级，明确其为 Flash 级）。

## 8. 安全 / 隐私

- **沙盒 cwd**：`agy` 的工作目录设为空目录；prompt 里意外的 `@相对路径` 只会落到沙盒，不触及项目文件。`@绝对路径` 仍可按需精确喂图。
- OAuth 凭证属敏感信息，勿写进任何可分享文件 / 仓库。
- 免费档数据使用政策以官方为准；介意训练用途者应据此选择计划。

## 9. 部署顺序（增量到主线之上）

1. 装 `agy`，`agy` 无参登录（OAuth）一次，`agy models` 确认可用。
2. 选默认主力模型（建议宽裕池）。
3. 把原 Gemini 群友的调用从 gemini CLI 换成 `agyGenerate`（§4）；prompt 拼装沿用原人设/历史逻辑。
4. 接 `--conversation` 记忆（§5）+ `convMap` 落盘。
5. 接思考链 → 观察模式（§6）。
6. 加 429 兜底（§7）。
7. 自测：长中文回复无 `�`；多轮记忆跨重启存活；思考块正常出现在观察模式；紧池撞墙能降级。
