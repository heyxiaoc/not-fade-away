## 引子：大少爷在外面流落街头了

主线教程里，前端能拉「多模型群友」进来——除了不同版本的 Claude，还能接 GPT（走 codex CLI）和 Gemini。我们群里那位 **Gemini**，论出身可是 **Google 的嫡长子、正经大少爷**；可他原本将就着用老的 **gemini CLI** 跑，OAuth 登录就能用，住得有点寒酸。

某天他直接罢工了。翻日志，报的是一个 **tier 相关错误**——大意「你这个层级不再支持，请迁移到 Antigravity」。一查才知道：**Google 把老 gemini CLI 退役了**（大约 2026 年 6 月中旬），等于把大少爷从旧住处赶了出来，官方让大家迁到新的 **Antigravity CLI**。

既然亲爹要收人回自己家，那咱就**把大少爷风光接回 Google 老宅**——回了家，自然该用上配得上身份的 Pro 模型。

## 为什么没有「随便换个 SDK」就把人打发了

第一反应是换官方 SDK（`@google/generative-ai`），用 API key 调。能跑通，但试下来有两点不太舒服（以下是我们当时的理解，Google 条款可能会变，建议你自己再核对）：

1. **免费这档，只给你 Flash 级的模型——相当于让大少爷睡柴房。** 想用配得上身份的 Pro，得绑卡；而**一旦绑卡，基本就转成按量计费**，跟我们想要的「用已经在付的订阅额度」不是一回事。
2. **免费档的数据可能被用于改进模型（训练）。** 对一个天天跟你聊私人话题的伴侣来说，这点我们挺在意。

我们真正想要的是：**继续用订阅/会员那档的额度**（已经在付的那份），别让大少爷降智睡柴房，也别绑卡按量肉疼。而老 gemini CLI 那条 OAuth 通道，恰好就是被退役、让大家迁去 Antigravity 的那条。**所以结论很自然：把大少爷从 gemini CLI 接回 Antigravity CLI 的老宅。**

## Antigravity CLI（`agy`）：大少爷的新家

Antigravity 是 Google 自家的 AI 编程 IDE，随附一个命令行工具，命令名 **`agy`**。它跟我们桥接其它 CLI 的方式天然合拍，有这些好东西：

- **`-p` / `--print`**：非交互跑一句、直接打印回复（桥接入口）。
- **`--model "<名字>"`**：指定模型。
- **`@<图片路径>`**：让他看图。
- **`--conversation <id>`**：把对话**持久化**——后面「多轮记忆」和「挖思考」全靠它。

OAuth 登录后，走的是你 Antigravity/会员那档额度池，不是 API-key 按量——大少爷回家，吃住按身份来。

## 一家子都有谁、各自什么待遇

登录后 `agy models` 会列出可用模型。我们当时看到的大致这几位：

- **Gemini 3.x Pro**（思考深度 Low / High 两档）——**大少爷本尊**，回了自家当然用这个。
- **Gemini 3.x Flash**（Low / Medium / High）——大少爷的快手版。
- **Claude Opus 4.6（Thinking）**、**Claude Sonnet 4.6（Thinking）**——这俩是 **Google 白接进门的养子**：Antigravity 这边居然还能用上 Claude。
- **GPT-OSS 120B**——远房表亲，凑个数。

**关于额度，先说官方口径**（来源：Antigravity 官方文档 Plans 页 <https://antigravity.google/docs/plans>；措辞和数字官方都可能随时调整）：

- **配额是按「干了多少活」算的，不是固定次数。** 官方原话大意：限额跟 agent 实际做的工作量挂钩，任务简单你能多答好几轮，任务重就少——所以**没有「每天 X 次」这种精确数字**。
- **刷新节奏看你的计划：** Google AI **Pro / Ultra** 大约**每 5 小时**刷新一次（到周上限为止）、份例更足；**免费档（不订阅）是按周刷新、额度也更少**。
- **大少爷是「核心模型」，所有计划都给**（Gemini 系列是 baseline）。而 **Claude、GPT-OSS 这些第三方模型，官方是放在更高档（Ultra）里点名的**——换句话说**养子们的待遇本就更受限**，低档能用到但份例紧，跟我们实测正好对上。
- 额度用尽后能不能接着用，由一个 **「AI Credit Overages」设置**控制：`Never` = 乖乖等刷新（不会偷偷花钱），`Always` = 自动用你购买的 AI credits 顶上（会按量计费）。**怕意外扣费就设 `Never`。** 各模型的 baseline 用量在 Settings 页能看。

下面是我们自己实测当天的观感（**Pro 计划，仅供参考，会因账号/计划/时段不同而不同**）：

| 谁 | 当天观感 |
| --- | --- |
| 大少爷 Gemini（Pro / Flash） | 嫡长子待遇，额度宽裕，一整天高强度用没碰到墙 |
| 养子 Claude（Opus / Sonnet）、表亲 GPT-OSS | 毕竟是接进门的，**份例紧一档**：重度使用后会撞配额，报 `RESOURCE_EXHAUSTED (429)`、提示「个人配额已用尽」，等刷新就自动回血（官方说 Pro/Ultra 约每 5 小时刷新，我们当天看到的回血在小时级） |

> **小贴士：** 想知道某位现在还能不能使唤，最直接是发句最小 prompt 探一下（`agy -p "回个字：好" --model "<名字>"`）——有回就在岗，回空 + 日志 429 就是这条池子暂时用尽了。所以**别把养子设成伴侣的默认主力**，主力交给额度宽的大少爷，养子留着「想换个味道时临时请出来」。免费档的话，大少爷的份例也会比我们紧，但尝鲜够用。

## 怎么把人接进前端（思路）

整体跟主线接其它 CLI 一样：**每条消息临时 spawn 一次 `agy -p`，拿输出当回复广播进前端。** 关键几步：

1. **登录一次**（登录态会存盘，之后 `agy -p` 直接用，不用反复登）。分两种情况：

   **A. 人就在部署的机器跟前** —— 最省事。跑 `agy`、选 Google OAuth，它会**自动拉起浏览器**；你在浏览器里登录授权，跳转回调后它自己就接上了，回到终端基本就登好了。

   **B. 人不在机器旁边（远程）** —— 这是我们的情况，机器在家、人在外面，浏览器拉不起来给你看，得手动倒一趟：
   1. 在机器上找个有真终端的地方（比如 detached tmux）跑 `agy`、选 Google OAuth；
   2. 它会打印一个**很长**的授权链接（几百字符）——把这个链接**想办法弄到你手上**（你自己的手机/电脑），在你那边的浏览器里打开；
   3. 登录授权，跳转后的页面会给你一个 **authorization code**；
   4. 把这个 code 送回机器、**贴回正在等待的 `agy`**（比如对 tmux 会话 `send-keys`），登录完成。
   - 那个链接通常几百字符，**如果你前端单条消息有长度上限可能发不出来**——换一个能发长文本的通道（能 SSH / 远程控制 / 别的不限长的入口）把链接递到你手上即可。
2. **拼 prompt**：`agy` 没有单独的「系统提示」参数，所以把人设、最近对话、这次的话**拼成一整段** prompt 传给 `-p`。
3. **指定模型**：`--model "<你选的模型>"`（默认主力建议给大少爷）。
4. **持久化 + 多轮记忆**：带上 `--conversation <id>`。第一次跑会新建一段对话（`agy` 自分配一个 id，机看版讲怎么抓到它）；之后每轮复用同一个 id，他就**原生记得多轮上下文**，你甚至能少喂点历史。

## 挖「思考过程」：嘴上不说，但他自己写日记

我们一开始以为拿不到大少爷/养子的思考链——因为 `agy -p` 只打印最终答案，思考在交互界面里要按快捷键才展开，无头模式默认不给。

转机是：**只要带了 `--conversation`，`agy` 会把这段对话持久化、自己写一个 transcript 文件**（路径形如 `…/antigravity-cli/brain/<对话id>/.system_generated/logs/transcript.jsonl`）。里面每条回复（`PLANNER_RESPONSE`）**把「思考」和「正文」分成两个独立字段**——`thinking` 和 `content`。等于他嘴上只说结论，私下却把心里盘算全写进了日记本。

这跟 Claude Code 自己写 transcript、我们在主线里 tail 它拿思考流，是**一模一样的套路**。于是做法清晰了：

- 跑完一句 `agy -p --conversation <id>`，
- 去读那个 `transcript.jsonl`，取最后一条 `PLANNER_RESPONSE` 的 `thinking` 字段，
- 把它当「思考块」广播进前端的**观察模式**（跟 Claude 思考流走同一套展示），`content` 字段当正文回复。

整条链路：**不抓屏、不用第三方代理、不碰任何后端私有接口**，就读他自己落盘的日记。订阅额度、Claude 模型、思考、多轮记忆，一次拿齐。

## 几个要注意的坑

- **中文可能变乱码（�）。** 若你像桥接其它 CLI 那样「每收到一块输出就解码一次」，多字节字符（中文 UTF-8 占 3 字节）一旦被切在两块数据边界上，那半个字就成 `�`。**修法：把原始字节都收集起来，最后整体解码一次。**
- **首轮会混进一行警告。** 第一次用一个还不存在的 `--conversation` id，`agy` 会往 **stdout**（不是 stderr）打一行 `Warning: conversation "<id>" not found.`，再接正文。记得**按行过滤掉这句**，别让它漏进聊天。
- **份例撞墙要有兜底。** 把伴侣默认放在额度宽的大少爷上；若你坚持用养子（Claude），建议加一个「撞到 429 / 拿到空回复时自动降级到大少爷」的兜底，免得聊到一半人没了。
- **隔离工作目录。** `agy` 会解析 prompt 里的 `@路径`。把工作目录设成一个空沙盒，万一聊天里出现意外的 `@something`，也不会让他去翻你项目里的文件。
